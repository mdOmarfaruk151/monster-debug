1. Difference Between Local Storage and Session Storage ?

No. Local Storage Session Storage
1 Their main difference is that local storage data does not expire. Session storage data gets cleared when the user closes its browser window.
2 Local storage is not session-based, all the data must be deleted via javascript or manually. Page sessions are valid for only one tab at a time. So, if a user has three tabs open, each one will have its own page session.
3 local storage exists until explicitly deleted, you can use it for data you want to store long term, even after the user has closed the tab or browser. Session storage is only valid for one tab, you would use it when you want to store data that should not be accessible from other tabs.
3 For example, if you have a user setting you want to preserve, such as the “remember me” toggle. For example, if you have a ticket purchasing flow on your website, you would want to use session storage so that the user can only purchase one ticket per tab, and the two separate ticket purchasing tabs do not interfere with each other.
3 Use local storage when you want to share data between tabs or windows. So, if you have a shopping cart on your website and you want the user to be able to access their cart from any tab, you would need to use local storage. Session storage is a good candidate for temporary data that you want to store to improve the user experience or performance of your website.
